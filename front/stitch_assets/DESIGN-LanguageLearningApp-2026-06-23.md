# DESIGN.md — Дизайн-система и спецификация экранов

**Продукт:** AI Language Learning Platform
**Подход:** Dark-first, минимализм, AI-native ощущение
**Референсы:** gemini.google.com (2026 redesign), grok.com, labs.google/fx/tools/flow, Google Stitch design language

---

## 0. Из чего собран этот документ

Перед тем как описывать экраны, я изучил:

- **Google Stitch** — официальный AI-инструмент дизайна (stitch.withgoogle.com) и его подход к файлу `DESIGN.md`: машинно-читаемая дизайн-система, которая документирует токены и правила так, чтобы ей могли следовать и человек, и AI-агент при дальнейшей разработке. Этот документ построен по той же логике.
- **Gemini (редизайн 2026)** — переход к глубоким чёрным тонам (deep black, не серый), пульсирующий градиентный фон во время обработки запроса, минималистичная иконография, единая кнопка "+" вместо нагромождения меню, генеративные элементы (timeline/карты) встроенные прямо в диалог.
- **Grok.com / x.ai** — характерный для 2026 года стиль: тёмный фон, высокая контрастность текста, мало декора, акцент через один яркий цвет, плотная, но "дышащая" типографика.
- **Google Flow (labs.google/fx/tools/flow)** — бесконечный канвас, плавающие панели инструментов поверх тёмного фона, контент (видео/изображения) — главный герой экрана, интерфейс управления почти невидим, пока не нужен.
- **Общие тренды UI 2026** (подтверждено несколькими независимыми источниками): "dark-first" дизайн — тёмная тема проектируется первой, а не как инверсия светлой; тонкий glassmorphism (полупрозрачные слои с лёгким размытием) вместо тяжёлого блюра 2022 года; OLED-true-black фоны для энергоэффективности и контраста; variable fonts; AI-интерфейсы, которые "спрашивают, прежде чем отвечать" — то есть прозрачность и контроль пользователя как принцип, а не просто красивый чат-пузырь.

**Вывод для нашего продукта:** мы не копируем ни один продукт буквально — мы используем общий язык этой категории (AI-инструмент, конверсационный, тёмный, спокойный) и выражаем его через собственный сигнатурный элемент (см. ниже).

## 1. Дизайн-токены (Design Tokens)

### 1.1 Цветовая палитра (Dark mode — основной, единственный режим в MVP)

| Токен | HEX | Назначение |
|---|---|---|
| `--bg-base` | `#0A0A0C` | Основной фон приложения — почти чистый чёрный (OLED-true-black), не серый |
| `--bg-surface` | `#15151A` | Поверхность карточек, панелей, модалок — на один уровень выше базы |
| `--bg-surface-raised` | `#1E1E24` | Приподнятые элементы: активные карточки, hover-состояния, поля ввода |
| `--border-subtle` | `#2A2A32` | Тонкие разделители, контуры карточек (1px, никогда не ярче содержимого) |
| `--text-primary` | `#F5F5F7` | Основной текст — не чистый белый (избегаем избыточного контраста, как в Grok/Gemini) |
| `--text-secondary` | `#9A9AA5` | Вторичный текст, подписи, метаданные |
| `--text-tertiary` | `#62626C` | Плейсхолдеры, неактивные элементы |
| `--accent-primary` | `#6E5BFF` | Сигнатурный фиолетово-индиго акцент — единственный "громкий" цвет в системе (см. сигнатуру ниже) |
| `--accent-glow` | `#8B7CFF` | Более светлый вариант акцента — для свечения (glow), градиентов, состояний фокуса |
| `--success` | `#3DD68C` | Правильный ответ, завершённый урок, рост streak |
| `--warning` | `#E8B339` | Близко к лимиту (например, AI Speaking 5 мин/день) |
| `--error` | `#FF5C6C` | Ошибка, неправильный ответ (используется приглушённо, не агрессивно — это обучающая ошибка, не сбой системы) |

**Принцип:** один акцентный цвет (`--accent-primary`), а не радужная палитра. Это сознательный выбор — в отличие от "четырёхцветной эры" старого Google-дизайна, наш продукт держит дисциплину одного цвета, как Grok и текущий Gemini, чтобы интерфейс не визуально соревновался с обучающим контентом.

### 1.2 Типографика

- **Display / заголовки:** Geometric sans с чуть зауженными формами (например, в духе Söhne/General Sans) — используется крупно и редко: только на онбординге, на заголовках разделов, на достижениях. Не использовать на каждом экране — заголовок должен быть событием, а не рутиной.
- **Body / основной текст:** Нейтральный, максимально читаемый sans (Inter-подобный) — используется везде, где пользователь читает объяснения, словарь, чат. Поддержка кириллицы (Russian) и таджикской/латинской/кириллической письменности (Tajik пишется кириллицей) обязательна — шрифт должен одинаково хорошо выглядеть на всех 3 UI-языках.
- **Variable font подход:** один файл шрифта с гибкой осью weight (300–700) вместо нескольких статических файлов — снижает вес страницы, позволяет тонко настраивать иерархию (например, активное слово в словаре чуть тяжелее окружающего текста).
- **Числа/метрики (XP, streak, баллы экзаменов):** табличные цифры (tabular numerals), чтобы цифры не "дрожали" при обновлении счётчиков в реальном времени.

### 1.3 Форма и поверхности

- **Border-radius:** средний (12–16px на карточках, 8px на кнопках/полях) — не острые углы (слишком "корпоративно"), не подчёркнуто круглые (слишком "игрушечно" для платформы, где есть взрослые цели вроде IELTS).
- **Glassmorphism — точечно, не везде:** полупрозрачные слои с лёгким backdrop-blur используются только для **плавающих элементов поверх контента** — floating-кнопка AI Tutor, верхняя панель навигации при скролле, модальные окна. Обычные карточки урока — **плоские**, без блюра, чтобы не отвлекать от учебного контента (это разделение — "глубина для навигации, плоскость для контента" — ключевой структурный принцип всей системы).
- **Тени:** почти не используются в классическом виде (blur+offset). Вместо теней — разница в `--bg-surface` уровнях и тонкая `border-subtle`. Это держит интерфейс "плоским и спокойным", в духе Flow/Grok, а не "плавающим" в духе iOS skeuomorphism.

### 1.4 Сигнатурный элемент (Signature)

**"Living Accent Line"** — тонкая (2px) градиентная линия `--accent-primary → --accent-glow`, которая появляется только в одном месте за раз и обозначает "точку активности AI прямо сейчас": под полем ввода AI Tutor, пока ИИ формирует ответ; вокруг аватара во время Speaking-сессии; как прогресс-бар во время генерации урока. Она **пульсирует** медленно (не мигает), отражая идею "ИИ сейчас думает/работает" — наш ответ на пульсирующий градиентный фон Gemini, но выраженный сдержаннее и компактнее, чтобы не доминировать на образовательном контенте.

### 1.5 Движение (Motion)

- Переходы между экранами — fade + едва заметный vertical shift (8px), 200–250ms, ease-out. Никаких bounce/spring-эффектов — это снижает ощущение "несерьёзности" для аудитории, готовящейся к IELTS/работе.
- Появление AI-ответа в чате — текст стримится по словам (как у Gemini/Grok), а не появляется целым блоком — критично для ощущения "живого" общения.
- Достижения (Achievement unlock) — единственное место, где допустима более выразительная анимация (лёгкое масштабирование + свечение бейджа) — это намеренный момент вознаграждения, контраст с общей сдержанностью.
- Уважение `prefers-reduced-motion` — обязательно на всех анимациях.

## 2. Спецификация экранов

> Для каждого экрана: назначение, структура, ключевые UI-паттерны, как используются токены выше.

### 2.1 Онбординг: выбор языка интерфейса

**Назначение:** первое впечатление от продукта — должно сразу транслировать "это современный AI-инструмент", а не "очередное приложение для изучения языков".

**Структура:**
- Полноэкранный тёмный фон `--bg-base`, по центру — три крупные карточки-плитки (Tajik / Russian / English), каждая с названием языка, написанным на самом этом языке (например, "Тоҳикӣ" не "Tajik")
- Лёгкий ambient-градиент на фоне (очень приглушённый, на грани заметности) — отсылка к пульсирующему фону Gemini, но статичный, чтобы не отвлекать на первом экране
- Никакого логотипа-анимации при загрузке — сразу к делу, в духе "AI-инструмента", а не маркетингового сайта

**Паттерн:** карточки увеличиваются (scale 1.02) и подсвечиваются `--accent-glow` рамкой при hover/focus — единственная интерактивность на экране

---

### 2.2 Онбординг: Placement Test

**Назначение:** определить уровень без ощущения "экзамена-стресса"

**Структура:**
- Минималистичный progress-индикатор сверху — не классический progress-bar, а ряд тонких сегментов (по числу вопросов), заполняющихся `--accent-primary` по мере прохождения
- Один вопрос на экран, крупный текст вопроса (display-шрифт), варианты ответа — карточки в один столбец на mobile-ширине, даже на web (мы намеренно не используем широкую раскладку — концентрация на одном вопросе важнее плотности информации)
- Никакого таймера на экране (тест — не стресс-тест на скорость, хотя сложность адаптивна внутри)
- В конце — экран результата: крупно показан CEFR-уровень (например, "B1") с короткой человеческой формулировкой ("Ты уверенно общаешься на базовые темы и готов к более сложной грамматике") — никаких сухих процентов на первом экране результата

---

### 2.3 Онбординг: Цели обучения

**Назначение:** выбор 1–3 целей из списка (Travel, Work, IELTS и т.д.)

**Структура:**
- Сетка карточек-чипов (bento-style grid, без строгой сетки 3×3 — допустимы карточки разного размера для целей с иконками разной сложности, как в bento-grid тренде 2026)
- Каждая цель — иконка (линейная, монохромная, в стиле текущей минималистичной иконографии Google) + название
- Выбранные карточки получают `--accent-primary` бордер и слегка приподнятый `--bg-surface-raised` фон — без чекбоксов, сама карточка визуально "включается"

---

### 2.4 Главный экран (Dashboard)

**Назначение:** центр управления — что делать сегодня. Должен отвечать на вопрос "что мне открыть прямо сейчас" за 2 секунды.

**Структура (сверху вниз):**
1. **Верхняя панель** — полупрозрачная (glassmorphism), фиксированная при скролле: аватар/streak счётчик слева, XP справа, без лишнего лого по центру
2. **Hero-блок "Продолжить обучение"** — самая крупная карточка экрана, показывает текущий незавершённый урок с превью темы и кнопкой продолжения. Это "thesis" экрана — то самое характерное действие, которое определяет ценность продукта (по аналогии с принципом "hero как тезис" в современном вебдизайне)
3. **Карточка "Повторение"** (Spaced Repetition queue) — компактная, показывает число элементов к повторению сегодня, с лёгкой `--warning` точкой-индикатором, если накопилось много несделанных повторений
4. **Ряд быстрых действий** — горизонтальный скролл из небольших карточек: AI Tutor, Real World Missions, Exams — каждая с минималистичной иконкой и без лишнего текста
5. **Нижний блок — Streak/XP полоса** — тонкая горизонтальная полоса прогресса до следующего уровня геймификации, без отдельного "игрового" визуального языка (не яркие значки/монетки — стиль должен оставаться сдержанным даже в игровых элементах)

**Ключевой паттерн:** *никаких модальных окон при заходе* (никаких popup "Не забудьте позаниматься!") — вся мотивация выражена через спокойную визуальную иерархию dashboard, а не через прерывания

### 2.5 Экран урока (Lesson)

**Назначение:** последовательное прохождение блоков урока (теория → словарь → упражнения → тест)

**Структура:**
- Полноэкранный режим, верхняя панель сведена к минимуму: только прогресс текущего урока (тонкая линия) и кнопка выхода
- Один блок контента на экран, переход свайпом/кнопкой "Далее" — никакой длинной скроллящейся страницы со всем уроком сразу (повышает ощущение fokуса, снижает когнитивную нагрузку)
- **Блок теории:** текст на `ui_language` пользователя, ключевые грамматические конструкции выделены `--accent-primary` (не подчёркиванием/курсивом — цветом, без декоративности)
- **Блок словаря:** карточки слов в духе flashcard — слово крупно, перевод и транскрипция появляются по тапу/клику (контролируемое раскрытие информации, а не всё сразу)
- **Упражнения:** варианты ответа — крупные тактильные карточки (не радио-кнопки) — правильный ответ подсвечивается `--success`, неправильный — приглушённым `--error` с мгновенным, спокойным объяснением снизу (без резких "крестиков" и тряски — ошибка это часть обучения, а не сбой)
- **Финальный тест урока:** аналогичный паттерн, в конце — экран завершения с XP-начислением, показанным как короткая, сдержанная анимация числа (+15 XP), не конфетти

---

### 2.6 AI Live Tutor (чат)

**Назначение:** мгновенный доступ к ИИ-репетитору в любой момент

**Структура:**
- Доступен как floating-кнопка (glassmorphism, нижний правый угол на web) — открывается как панель, выезжающая справа (не полноэкранный модал — пользователь не должен терять контекст урока, если открыл тьютора во время прохождения урока)
- Чат-лента: сообщения пользователя — справа, `--bg-surface-raised` карточки; ответы ИИ — слева, **без пузыря-карточки**, просто текст на фоне `--bg-base` (характерный паттерн Gemini/Grok 2026 — ответ ИИ воспринимается как часть интерфейса, а не "сообщение от собеседника в пузыре")
- Во время генерации ответа — **Living Accent Line** (сигнатурный элемент, см. 1.4) пульсирует под полем ввода вместо классических "тёр точки" (•••)
- Поле ввода — нижняя панель, с быстрыми действиями над ним (чипы: "Объясни правило", "Дай пример", "Дай упражнение") — контекстные подсказки, не статичное меню

### 2.7 Real World Missions

**Назначение:** ролевой диалог с ИИ в жизненной ситуации

**Структура:**
- Экран выбора миссии: карточки-сценарии с короткой иллюстративной иконкой (линейная, монохром + акцент) и названием ситуации ("Заказать еду в ресторане"), отфильтрованные по целям/уровню пользователя
- Экран самого диалога — визуально идентичен AI Tutor чату (переиспользование паттерна — пользователь не должен переучиваться) с одним отличием: верхняя панель показывает "роль" ИИ в этой миссии (например, "Официант" вместо "AI Tutor")
- По завершении — экран фидбека: не просто "оценка 7/10", а структурированная карточка "Что получилось хорошо" / "Что можно сказать иначе" с конкретными примерами фраз — без избыточной похвалы или критики, нейтральный, конструктивный тон

---

### 2.8 AI Writing Exam

**Назначение:** письменный экзамен с детальной оценкой

**Структура:**
- Экран ввода — тема экзамена крупно сверху, под ней просто текстовое поле на весь оставшийся экран (никакого визуального шума — пользователь должен сфокусироваться на письме), счётчик слов внизу мелким текстом
- Экран результата — оценка представлена как **горизонтальные полосы-метрики** (grammar / vocabulary / cohesion / naturalness / style), каждая полоса заполнена `--accent-primary`, не "звёздочки" и не проценты крупным шрифтом — спокойная, аналитическая визуализация в духе дашборда, а не геймифицированная оценка
- Под метриками — текстовые рекомендации с конкретными примерами исправлений (оригинал → исправленный вариант, показанные как diff: удалённый текст приглушён и зачёркнут, добавленный — подсвечен `--success`)

---

### 2.9 AI Listening Exam

**Назначение:** аудирование с вопросами на понимание

**Структура:**
- Экран прослушивания — почти пустой, в центре — крупный, единственный элемент управления воспроизведением (play/pause), без визуализации waveform (чтобы не провоцировать "чтение" аудио визуально вместо слушания)
- Транскрипт скрыт нарочито — вместо текста место занимает спокойная ambient-анимация (едва заметная пульсация фона в `--accent-glow`, синхронизированная с воспроизведением — повторение мотива "AI работает" из 1.4, но в момент, когда работает не ИИ, а сам процесс слушания)
- Вопросы — отдельный экран после прослушивания, тот же паттерн карточек ответов, что в обычном уроке (3.5) — консистентность паттернов через весь продукт

---

### 2.10 AI Speaking (голосовая сессия, в рамках 5 мин/день лимита)

**Назначение:** голосовое взаимодействие с ИИ — самый "premium" по ощущению экран в продукте

**Структура:**
- Полноэкранный, максимально минималистичный: по центру — крупный круглый индикатор (аватар/орб), который пульсирует **Living Accent Line**-градиентом синхронно с речью ИИ (вдохновлено орбами голосовых интерфейсов Gemini Live, но в нашей фирменной фиолетово-индиго палитре, не дефолтный синий)
- Таймер дневного лимита — едва заметный, в углу, текстом ("3:42 осталось сегодня"), меняет цвет на `--warning` в последнюю минуту — без агрессивных предупреждений, спокойное информирование
- Транскрипт разговора — опционально скрываем по умолчанию (фокус на устной речи), можно развернуть свайпом снизу

### 2.11 Progress / Profile

**Назначение:** прогресс, достижения, еженедельные отчёты AI Coach

**Структура:**
- Верхний блок — крупная статистика (streak, total XP, уровень) в виде нескольких компактных карточек bento-grid (разного размера по значимости метрики — streak крупнее, чем менее важные счётчики)
- **AI Coach еженедельный отчёт** — отдельная карточка-письмо, оформленная как личное сообщение от ИИ (не дашборд-виджет): заголовок недели, короткий абзац от первого лица ("На этой неделе ты заметно лучше справлялся с Present Perfect..."), внизу — явный разделённый блок "Рекомендации на следующую неделю"
- **Achievements** — горизонтальная галерея бейджей, неразблокированные — показаны приглушённым силуэтом (`--text-tertiary` контур, без заливки) — мотивирующая "недостача", а не скрытие
- **История ошибок** (из Error Correction Engine) — компактный список повторяющихся ошибок с количеством повторов, без излишней детализации на этом экране — это сводка, не полный лог

---

### 2.12 Настройки

**Назначение:** язык интерфейса, цели, лимиты, аккаунт

**Структура:**
- Простой список секций (Account, Learning Goals, Interface Language, Notifications) — `--bg-surface` карточки-строки с шевроном справа
- Никакой переусложнённости — это утилитарный экран, единственный во всём продукте, где допустим чисто "табличный", не творческий подход — настройки должны быть предсказуемыми, а не "интересными"

## 3. Сквозные принципы (применимы ко всем экранам)

1. **Глубина только для навигации, плоскость для контента.** Glassmorphism/блюр — только на элементах управления над контентом (верхние панели, floating-кнопки, модалки). Сам учебный контент — всегда плоский и читаемый.
2. **Один акцентный цвет.** `--accent-primary` — единственный "громкий" цвет в системе. Success/warning/error используются функционально и приглушённо, не как часть декоративной палитры.
3. **AI говорит без пузыря.** Ответы ИИ в любом чат-подобном интерфейсе (Tutor, Missions) визуально слиты с фоном экрана — это сигнализирует "это часть продукта", а не "сообщение от третьего лица".
4. **Сигнатурная пульсация = "ИИ сейчас работает".** Единый визуальный язык (Living Accent Line) используется во всех точках, где система генерирует что-то в реальном времени — это создаёт целостное ощущение "живого" AI-продукта через весь интерфейс, а не только в чате.
5. **Ошибка — часть обучения, не сбой.** Везде, где показывается неправильный ответ пользователя — тон визуальный и текстовый спокойный, конструктивный, без тревожной красноты или резких эффектов.
6. **Один контент-блок на экран при обучении.** В отличие от dashboard (где допустима плотность), экран урока/упражнения/чата всегда сфокусирован на одной задаче за раз.
7. **Поддержка трёх UI-языков как дизайн-требование, не перевод-постфактум.** Все текстовые компоненты должны быть проверены на длину текста в Tajik/Russian (которые в среднем длиннее английского) — никаких жёстко зафиксированных по ширине текстовых контейнеров на кнопках/чипах.
